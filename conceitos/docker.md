# O que é Docker?
- é uma ferramenta que permite empacotar uma aplicação junto com tudo o que ela precisa para rodar dentro de algo chamado container. 

# O que é um container?
é como uma caixinha isolada que contém:
- seu código;
- dependências (bibliotecas, frameworks);
- configurações;
- ambiente (Node, Python).

# Docker resolve:
- inconsistência de ambiente;
- setup complicado;
- conflitos de versão;
- problemas de deploy.

# Detalhes
- diferentemente de vm´s, o Docker é mais rápido e mais leve porque compartilha o kernel da máquina, e não o SO completo.

# Imagem de container
- é um arquivo estático, que funciona como um modelo ou receita contendo todas as instruções e configurações necessárias para
executar uma aplicação.

# Container docker
- é a instância em execução de uma imagem docker, representando o software funcionando em um ambiente isolado.

# Docker Hub
- é um repositório de imagens Docker;
- ele serve para baixar imagens prontas, compartilhar aplicações e padronizar ambientes.
